創夾資訊一群熱愛著資訊且平均年齡不到30 歲的年輕人所創立的資訊工作室，我們於 2016 年晚秋正式成立，正如同工作室名稱我們認為專案即為一個資料夾的概念，我們以新穎的架構以及紮實的技術豐富這個資料夾，最後將此資料夾封裝並且交給客戶。每個專案我們均採用 Git 進行版本控制保障每個研發人員皆可以快速的上手進行研發，並且採用 Travis CI 以及 Docker 以保障那些容易被其他團隊忽略的 CI/CD 持續整合/交付的部分。


軟硬體整合


秉持著與時俱進並且以技術作為研發的優先考量，採用著集結社群力量的開放原始碼的框架進行 Web / App / 資料挖掘 / IoT 的研發，並且提供完善的技術支援以及售後維運。我們擅長的程式語言以 Python , JavaScript 以及 C# 為主，包括著全端 MVC (Flask/Django) 框架 ，資料挖掘的 Kaggle 事件驅動，以 Raspberry PI 為主的 IoT 研發，以及跨平臺行動技術的掌握，並且搭配新形態的 NoSQL 資料庫例如 MongoDB / DocumentDB 進行整合性系統之實作，且我們在伺服器虛擬化例如 ESXi , XEN 以及 OpenStack 的規劃部屬已經有多年的經驗，主流的 AWS , Azure 以及 Bluemix 雲平臺部屬也駕輕就熟。

解決方案

我們採用 Python 作為我們的研發主要語言，並且充分利用 Python 在後端處理/資料處理的易用優勢。

> 在全端以及跨平臺的應用上，我們依照每一個專案的特性選擇採用 Flask 框架或基於 Flask 自行研發的 mu_Flask 框架，在跨平臺的領域上使用 RESTful API 的形態進行多種平臺應用的後端結合，而在 ORM 的部分依照專案規模會選擇採用傳統式的 SQLAlchemy 或是 NoSQL 的 MongoEngine 。

> 在資料採集，數據統計的應用上，我們會利用 numpy/pandas 設計相關的統計或是運算單元程式，並且利用 OpenMP / Dask 等分散式運算模塊完成大量數據統計運算的需求。最後在搭配相關的前後端框架，以及 matplotlib 進行可視覺化數據的呈現。

我們採用 Travis CI ，這個在開源界相對年輕的 CI/CD 工具。

雖然這個工具以及配套相對年輕，但是擁有著高可用性以及集中形態的部屬設定優勢且能夠在異環境多版本快速的進行部屬測試，能夠讓我們安全且快速的進行持續整合持續交付。並且在專案研發時我們能透過 Travis CI 保障異常事件能夠讓跟此專案有關的開發者在第一時間知道，讓我們能夠節省許多不必要浪費的時間。
